# CSR, SSR y SSG - ¿Qué son y cuándo usarlos?

En este vídeo, aprenderás las diferencias entre Client side rendering (CSR), Server side rendering (SSR) y Static site generation (SSG) y cuándo es mejor usar cada uno.

## ¿Cómo funciona?

CSR, SSR y SSG son tres enfoques diferentes para renderizar aplicaciones web:

- **CSR (Client Side Rendering)**: La aplicación se renderiza en el navegador del cliente. Esto significa que el servidor envía un HTML básico y luego el JavaScript se encarga de renderizar la aplicación en el cliente. Es ideal para aplicaciones altamente interactivas.
- **SSR (Server Side Rendering)**: La aplicación se renderiza en el servidor y se envía al cliente como HTML completo. Esto mejora el rendimiento inicial y la indexación por parte de los motores de búsqueda. Es ideal para aplicaciones que requieren SEO y tiempos de carga rápidos.
- **SSG (Static Site Generation)**: La aplicación se genera como archivos estáticos en el momento de la construcción. Estos archivos se sirven directamente desde un servidor web, lo que mejora el rendimiento y la escalabilidad. Es ideal para sitios web con contenido estático que no cambia con frecuencia.
